## Installation

```bash
# npm
npm install -D unocss-preset-magicolor

# pnpm
pnpm add -D unocss-preset-magicolor

# bun
bun add -d unocss-preset-magicolor
```

Before using this preset, install and configure UnoCSS by following the official [UnoCSS integration guide](https://unocss.dev/integrations/). Keep the UnoCSS presets your project already uses, then add `presetMagicolor()`.

## Setup

Add the preset to your UnoCSS config:

```ts
// uno.config.ts
import { defineConfig, presetWind4 } from 'unocss';
import { presetMagicolor } from 'unocss-preset-magicolor';

export default defineConfig({
  presets: [
    presetWind4(),
    presetMagicolor(),
  ],
});
```

`unocss-preset-magicolor` does not replace UnoCSS colors. It adds an `mc-` color layer on top of `theme.colors`, default UnoCSS palettes, and color-related utility rules.

## Usage

Use `mc-` color utilities directly in your markup:

```html
<button class="rounded px-4 py-2 bg-mc-rose-445 c-white hover:bg-mc-rose-620">
  Magic color
</button>
```

Unlike fixed palette steps such as `rose-500` or `blue-600`, Magicolor accepts arbitrary numeric depths:

```html
<div class="bg-mc-rose-445 c-mc-blue-187 border border-mc-neutral-733">
  Arbitrary depth colors
</div>
```

Depth colors are resolved from the active UnoCSS theme palette or from the 50, 100, 200...950 palette generated by [`magic-color`](https://github.com/zyyv/magic-color), then emitted as `oklch(...)` custom properties. Only scanned color names and depths are generated, so unused variables are skipped.

## Color Sources

Magicolor utilities can read colors from several places.

### UnoCSS Theme Colors

Built-in UnoCSS colors and custom `theme.colors` entries work out of the box:

```ts
import { defineConfig, presetWind4 } from 'unocss';
import { presetMagicolor } from 'unocss-preset-magicolor';

export default defineConfig({
  presets: [presetWind4(), presetMagicolor()],
  theme: {
    colors: {
      grape: '#679512',
      wine: { red: '#9c1d1e' },
    },
  },
});
```

```html
<button class="border border-mc-wine-red-575 bg-mc-rose-445 px-8 py-4">
  <span class="c-mc-grape-160">Hello World!</span>
</button>
```

### Arbitrary CSS Colors

Use UnoCSS arbitrary value syntax for one-off colors:

```html
<div class="bg-mc-[rgb(12_22_33)]-220 c-mc-[#789411]-430">
  Arbitrary magic color
</div>
```

### Global Semantic Colors

Use `presetMagicolor({ colors })` for stable semantic aliases such as `primary`, `brand`, or `surface`:

```ts
import { defineConfig, presetWind4 } from 'unocss';
import { presetMagicolor } from 'unocss-preset-magicolor';

export default defineConfig({
  presets: [
    presetWind4({ dark: 'class' }),
    presetMagicolor({
      colors: {
        primary: { color: 'rose', lightnessReverse: true },
        brand: '#4f7bff',
      },
      dark: {
        primary: 'blue',
        brand: '#8ab4ff',
      },
    }),
  ],
});
```

```html
<button class="bg-mc-primary-457 c-white ring ring-mc-brand-630 hover:bg-mc-primary-620">
  Semantic color
</button>
```

`colors` defines light-mode aliases. `dark` defines matching dark-mode aliases. When `presetWind4` is present, Magicolor follows its dark mode setting, including `class`, `media`, and custom selectors. If no Wind4 dark mode can be read, it falls back to `.dark`.

CamelCase aliases are also available as kebab-case classes. For example, `brandPrimary` can be used as both `c-mc-brandPrimary-457` and `c-mc-brand-primary-457`. Avoid alias names ending in digits because trailing digits are parsed as compact depth syntax.

## Local Color Definitions

Define component-local color variables directly in classes with `mc-<name>_<source>`:

```html
<div class="mc-card_[#9c1d1e]">
  <button class="bg-mc-card-450 px-8 py-4 shadow shadow-mc-card-620 hover:bg-mc-card-575">
    <span class="c-mc-card-610">Hello World!</span>
  </button>
</div>
```

The definition class only declares the color source. Actual CSS variables are generated from usage classes such as `bg-mc-card-450`, `shadow-mc-card-620`, and `c-mc-card-610`.

The source can be a theme color, a global alias, a CSS color, or a source depth:

```html
<div class="mc-card_primary mc-badge_rose-620 mc-chip_[oklch(62%_0.2_20)]">
  <span class="bg-mc-card-457">Card</span>
  <span class="bg-mc-badge">Badge</span>
  <span class="c-mc-chip-300">Chip</span>
</div>
```

## Lightness Reverse

Use `mc-lr-*` when a color should read numeric depths in reverse. For example, `50` reads from `950`, `450` reads from `550`, and `500` stays `500`.

```html
<div class="mc-lr-card_rose">
  <button class="bg-mc-card-50 hover:bg-mc-card-450">
    Reversed local color
  </button>
</div>
```

You can also reverse an existing color name without redefining its source:

```html
<section class="mc-lr-primary">
  <button class="bg-mc-primary-80 c-white ring ring-mc-primary-230">
    Reversed primary
  </button>
</section>
```

Bare `mc-lr` reverses the configured or theme color names currently used in that selector:

```html
<section class="mc-lr">
  <button class="bg-mc-primary-80 c-white ring ring-mc-rose-230">
    Inverted depth scale
  </button>
</section>
```

## Runtime Color Switching

Use `updateMagicColor` to update already-generated magic color CSS variables at runtime. Passing `document.documentElement` updates the page theme. Passing a component root element updates only that local scope.

```vue
<script setup lang="ts">
import { updateMagicColor } from 'unocss-preset-magicolor/helper';

function changePrimary() {
  updateMagicColor({
    name: 'primary',
    color: 'rgb(79 123 255)',
    lightnessReverse: true,
    dom: document.documentElement,
  });
}
</script>

<template>
  <button class="bg-mc-primary-457 px-8 py-4 c-white hover:bg-mc-primary-620" @click="changePrimary">
    Change primary
  </button>
</template>
```

`updateMagicColor` reads existing variables such as `--mc-colors-primary-DEFAULT` and `--mc-colors-primary-457`, then updates only those variables. It does not infer new variables from DOM class names, so make sure UnoCSS has generated the variables first with classes such as `c-mc-primary` or `bg-mc-primary-457`.

## Preset Options

```ts
interface PresetMcOptions {
  colors?: Record<string, string | { color: string; lightnessReverse?: boolean }>
  dark?: Record<string, string | { color: string; lightnessReverse?: boolean }>
}
```

| Option | Description |
| --- | --- |
| `colors` | Global light-mode semantic color aliases emitted under `:root` when matching usages are scanned. |
| `dark` | Global dark-mode aliases for the same semantic names. These override source variables under the configured dark selector or media query. |
| `lightnessReverse` | Reverses numeric depth lookups for one alias. Base variables such as `DEFAULT` are not treated as `500`. |

## Layer Order

Magicolor utilities are emitted in the UnoCSS layer named `unocss-preset-magicolor` with default order `-100`. Higher layer orders are output later, so native UnoCSS utilities can override `mc-*` utilities by default when both set the same CSS property.

Override only the layer order when you want Magicolor utilities to win:

```ts
import { defineConfig, presetWind4 } from 'unocss';
import { presetMagicolor } from 'unocss-preset-magicolor';

export default defineConfig({
  presets: [presetWind4(), presetMagicolor()],
  layers: {
    'unocss-preset-magicolor': 1,
  },
});
```

## Compatibility Notes

- Normal UnoCSS variants such as `hover:`, `focus:`, `dark:`, and variant groups work with `mc-*` utilities.
- Shortcuts, safelist entries, attributify mode, `transformerDirectives`, and `transformerCompileClass` are covered by the preset's usage scanner.
- Special color keywords such as `transparent`, `currentColor`, and `inherit` are emitted directly without generating magic color variables.
- Colors explicitly configured in the UnoCSS `theme` take priority; missing depths are generated or interpolated by the preset.
